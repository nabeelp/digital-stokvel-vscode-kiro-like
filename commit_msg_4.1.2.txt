feat(group-service): integrate CBS client for group account creation (Task 4.1.2 ✅)

Integrates Core Banking System (CBS) client into Group Service to create real banking 
accounts when groups are created. Replaces simulated account number generation with 
actual CBS API calls, implementing the group creation flow defined in the design document.

**Task Completion:**
- ✅ Task 4.1.2: Implement create group savings account operation  
- Progress: 116/207 (56%) | Phase 4: 2/18 (11%)
- Design Reference: Section 8.1 - Core Banking System Integration, Group Creation Flow

**Implementation:**

1. **Group Service Integration (GroupService.cs):**
   - Added ICbsClient dependency injection
   - Replaced GenerateAccountNumber() with real CBS API call
   - Updated CreateGroupAsync flow:
     * Creates group and members in database
     * Calls CBS client CreateGroupAccountAsync(groupId, accountName)
     * Stores real CBS account number in GroupSavingsAccount table
     * Account name format: "STOKVEL-{GroupName}"
   - Comprehensive error handling and logging
   - Transaction integrity: Group creation fails if CBS call fails

2. **Error Handling:**
   - Catches CbsClientException from CBS client
   - Logs error with group ID and error message
   - Throws InvalidOperationException with user-friendly message
   - Error message: "Unable to create banking account for the group. Please try again later."
   - Preserves inner exception for debugging
   - Prevents orphaned groups without CBS accounts

3. **Dependency Injection (Program.cs):**
   - Added using directive for DigitalStokvel.Infrastructure
   - Registered CBS client: builder.Services.AddCbsClient(builder.Configuration)
   - CBS client configured from appsettings.json "CbsClient" section
   - Injected into Group Service constructor

4. **Project Dependencies:**
   - Updated DigitalStokvel.GroupService.csproj
   - Added project reference to DigitalStokvel.Infrastructure
   - Enables access to ICbsClient interface and CbsClientException

5. **Sequence Flow (per Design Document Section 8.1):**
   ```
   GroupService → Begin database transaction
   GroupService → Insert group record
   GroupService → Insert group_members (chairperson)
   GroupService → Call CBS: CreateGroupAccountAsync(groupId, accountName)
   CBS → Returns account number + details
   GroupService → Insert group_savings_account with CBS account number
   GroupService → Generate invitation tokens
   GroupService → Commit transaction
   ```
   - If CBS call fails, transaction rolls back (database not modified)
   - Group creation atomic: Either fully succeeds or fully fails

**Technical Notes:**

1. **Account Naming Convention:**
   - Format: "STOKVEL-{GroupName}"
   - Example: "STOKVEL-Monthly Savings Group"
   - Helps CBS identify stokvel accounts
   - Useful for CBS reporting and auditing

2. **Error Propagation:**
   - CbsClientException → InvalidOperationException
   - Technical details logged but not exposed to user
   - User sees friendly message
   - Inner exception preserved for support team

3. **Transaction Boundary:**
   - Database transaction in EF Core
   - CBS call within transaction boundary
   - If CBS fails, transaction rolls back
   - Prevents incomplete group creation

4. **CBS Client Resilience (from Task 4.1.1):**
   - Retry: 3 attempts with exponential backoff (2s, 4s, 8s)
   - Circuit breaker: Opens after 5 failures, 1-min break
   - Combined policy ensures robustness
   - GroupService benefits automatically
