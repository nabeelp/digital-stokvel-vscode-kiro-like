## Architectural Decisions

### 1. Single Activity Architecture
- Modern Android best practice
- Compose Navigation for screen transitions
- Simplified lifecycle management

### 2. MVVM Pattern
- ViewModel for business logic
- StateFlow for reactive UI updates
- Clear separation of concerns

### 3. Repository Pattern
- Abstract data sources (API + local database)
- Single source of truth (Room as cache)
- Offline-first architecture

### 4. Dependency Injection (Hilt)
- Compile-time dependency resolution
- Scoped lifecycle management
- Testability and modularity

### 5. Material Design 3
- Modern Android design language
- Dynamic color support (Android 12+)
- Adaptive layouts for tablets

### 6. Offline-First
- Room database as local cache
- Network monitor for connectivity changes
- Sync queue for offline operations
- WorkManager for background sync

### 7. Security by Design
- HTTPS-only (enforced by Network Security Config)
- EncryptedSharedPreferences for tokens
- Biometric authentication support
- ProGuard obfuscation for release

## Technology Justifications

### Kotlin + Jetpack Compose
- **Kotlin:** Modern, concise, null-safe language
- **Compose:** Declarative UI with less boilerplate
- **Material3:** Latest design system with dynamic color

### Hilt over Koin
- **Compile-time:** Catches errors at build time
- **Performance:** No reflection overhead
- **Android Integration:** First-class Android support

### Retrofit + Kotlinx Serialization
- **Retrofit:** Industry standard REST client
- **Serialization:** Kotlin-native, multiplatform-ready

### Room over Realm
- **Official:** Google's recommended solution
- **SQL:** Familiar query language
- **Testing:** Easy to mock with in-memory DB

### Coil over Glide/Picasso
- **Kotlin-first:** Native Kotlin API
- **Compose:** First-class Compose support
- **Modern:** Uses Kotlin coroutines
