COMPREHENSIVE REPORTING AND EXPORT SYSTEM

This commit implements a complete reporting and export system for the Digital Stokvel Web Dashboard, enabling Chairpersons, Treasurers, and Secretaries to generate, preview, and download various financial and operational reports. The implementation follows the reporting requirements specified in design.md and requirements.md (NFR-07, FR-08).

═══════════════════════════════════════════════════════════════════════

🎯 KEY FEATURES

1. **Multi-Format Report Generation**
   - PDF: Professional reports for AGM records and stakeholder presentations
   - CSV: Tabular data for spreadsheet analysis
   - Excel: Rich formatted reports with formulas and charts support
   - On-demand generation with customizable date ranges
   - Secure presigned URLs for temporary download access

2. **Comprehensive Report Types**
   - Contribution History: Member payment tracking with status summaries
   - Payout History: Disbursement records with approval chain
   - Member Activity: Engagement metrics, payment performance, streaks
   - Group Ledger: Complete transaction log (immutable audit trail)
   - Annual Summary: Year-end report with monthly breakdowns
   - Each report type with emoji icons for quick visual identification

3. **Date Range Filtering**
   - Predefined presets: This Month, Last Month, This Year, Last Year
   - Custom date range selection with date picker inputs
   - Flexible filtering for targeted report generation
   - Period display in generated report metadata

4. **Report History Management**
   - List of previously generated reports with metadata
   - Download links for all generated reports
   - Report details: type, format, size, generation date, period
   - One-click re-download from history
   - Expiration handling for presigned URLs

5. **POPIA Compliance - Data Export**
   - Right to Access: User can export all personal data
   - Right to Data Portability: JSON/PDF format export
   - Comprehensive data sections: profile, contributions, payouts, groups, votes, disputes
   - Secure export process with audit trail
   - Compliance with POPIA Section 23-25 requirements

6. **User Interface Design**
   - Split-view layout: Generator form (left) + History list (right)
   - Interactive format selection with radio buttons and icons
   - Date preset buttons for quick range selection
   - Real-time generation status with loading spinner
   - Success/error messaging with auto-dismiss
   - Empty state for first-time users
   - Responsive grid layout (2 columns desktop → 1 column mobile)

📊 API INTEGRATION

**Reporting Service Endpoints:**

1. **POST /v1/reports/generate**
   - Generate report with specified type, format, and filters
   - Body: { group_id, report_type, format, filters: { from_date, to_date, member_id?, status? } }
   - Returns: ReportMetadata with presigned file_url
   - Used for: On-demand report generation

2. **GET /v1/groups/:id/reports**
   - List generated reports for a group
   - Query params: page, page_size (pagination)
   - Returns: Paginated list of ReportMetadata
   - Used for: Report history display

3. **GET /v1/reports/:id**
   - Get single report metadata by ID
   - Returns: ReportMetadata with download URL
   - Used for: Report details and re-download

4. **GET /v1/reports/contribution-history**
   - Get contribution history data (for preview)
   - Query params: group_id, from_date?, to_date?
   - Returns: ContributionHistoryReport
   - Used for: Report data preview before generation

5. **GET /v1/reports/payout-history**
   - Get payout history data (for preview)
   - Query params: group_id, from_date?, to_date?
   - Returns: PayoutHistoryReport
   - Used for: Report data preview before generation

6. **GET /v1/reports/member-activity**
   - Get member activity data (for preview)
   - Query params: group_id, from_date?, to_date?
   - Returns: MemberActivityReport
   - Used for: Report data preview before generation

7. **GET /v1/reports/group-ledger**
   - Get group ledger data (for preview)
   - Query params: group_id, from_date?, to_date?
   - Returns: GroupLedgerReport
   - Used for: Immutable transaction history

8. **GET /v1/reports/annual-summary**
   - Get annual summary data (for preview)
   - Query params: group_id, year
   - Returns: AnnualSummaryReport
   - Used for: Year-end reporting

9. **POST /v1/users/me/data-export**
   - Export user's personal data (POPIA compliance)
   - Body: { format, include_sections[] }
   - Returns: ReportMetadata with export file URL
   - Used for: Right to Access, Right to Data Portability

**Report Metadata Structure:**
```json
{
  "id": "report-uuid",
  "group_id": "group-uuid",
  "report_type": "contribution_history",
  "format": "pdf",
  "filters": {
    "from_date": "2026-01-01",
    "to_date": "2026-03-31"
  },
  "generated_at": "2026-03-24T14:30:00Z",
  "generated_by": {
    "id": "user-uuid",
    "name": "Thandi Dlamini",
    "role": "chairperson"
  },
  "file_url": "https://storage.blob.core.windows.net/reports/...",
  "file_size_bytes": 524288,
  "expires_at": "2026-03-25T14:30:00Z"
}
```

🎨 UI/UX DESIGN DECISIONS

1. **Split-View Layout**
   - Rationale: Separate report generation from history browsing
   - Left panel: Report generator form (focused input area)
   - Right panel: Generated reports list (reference and re-download)
   - Benefits: Clear workflow separation, efficient space utilization

2. **Date Range Presets**
   - Rationale: Reduce friction for common reporting periods
   - Presets: This Month, Last Month, This Year, Last Year
   - Auto-calculate dates based on current date
   - Still allow custom date ranges for flexibility

3. **Emoji Icons for Report Types**
   - 💰 Contribution History: Money bag for payments
   - 📤 Payout History: Outbox for disbursements
   - 👥 Member Activity: People for engagement
   - 📊 Group Ledger: Chart for financial records
   - 📅 Annual Summary: Calendar for yearly reports
   - Rationale: Visual recognition, international understanding

4. **Format Radio Buttons with Icons**
   - PDF 📄: Document icon (professional reports)
   - CSV 📋: Clipboard icon (spreadsheet data)
   - Excel 📗: Green book icon (formatted workbooks)
   - Rationale: Clear visual distinction, tactile interaction

5. **Orange/Red Gradient for Reports Button**
   - Gradient: #F97316 (orange) → #EF4444 (red)
   - Rationale: Visual differentiation from other buttons
     * Green = Members (add, manage)
     * Blue = Contributions (view, track)
     * Purple = Payouts (approve, disburse)
     * Orange/Red = Reports (generate, export)
   - High contrast, attention-grabbing for important feature

6. **Report History Cards**
   - Card layout for each generated report
   - Metadata displayed: type, format, size, date, period
   - Download button always visible
   - Hover effects: lift animation + shadow
   - Rationale: Easy scanning, quick access, clear hierarchy

7. **Empty State Design**
   - Large folder emoji: 📁 (friendly, recognizable)
   - Two-line explanation:
     * "No reports generated yet"
     * "Generate your first report using the form above"
   - Rationale: Guide first-time users, reduce confusion

8. **Loading States**
   - Generate button shows spinner during generation
   - Button disabled during generation
   - Button text changes: "Generate & Download Report" → "Generating..."
   - Rationale: Clear feedback, prevent duplicate requests

9. **Success/Error Messaging**
   - Animated slide-in from top
   - Auto-dismiss after 5 seconds
   - Success: Green background + checkmark icon
   - Error: Red background + warning icon
   - Rationale: Non-intrusive feedback, no manual dismiss required

🔐 SECURITY & COMPLIANCE

**POPIA Compliance (Protection of Personal Information Act):**

1. **Right to Access (Section 23):**
   - exportUserData() provides complete personal data export
   - Includes: profile, contributions, payouts, groups, votes, disputes
   - Format options: PDF (human-readable), JSON (machine-readable)
   - Audit trail: Export logged with timestamp and user ID

2. **Right to Data Portability (Section 25):**
   - JSON format export for machine-readable data transfer
   - Structured format compatible with other systems
   - Complete data set without vendor lock-in

3. **Secure File Storage:**
   - Reports stored in Azure Blob Storage / AWS S3
   - Presigned URLs with expiration (24-hour TTL)
   - No direct file access without authentication
   - URL expires after download or time limit

4. **Audit Trail:**
   - All report generations logged with:
     * User ID and name
     * Report type and format
     * Date range and filters
     * Generation timestamp
   - Retention: 7 years (FICA compliance)

5. **Role-Based Access:**
   - Reports accessible to: Chairperson, Treasurer, Secretary
   - Members can only export their own data
   - Group-level reports require group membership
   - Token validation on all API requests

**Data Privacy:**

1. **Sensitive Data Handling:**
   - ID numbers encrypted in database
   - Phone numbers masked in reports (optional)
   - Bank account numbers never displayed in reports
   - PIN never stored or transmitted in plaintext

2. **Consent Management:**
   - Credit bureau reporting requires explicit opt-in
   - Data export shows privacy notice before generation
   - User can revoke consent at any time

3. **Retention Policy:**
   - Generated reports: 30 days in storage
   - After expiration: Reports deleted, metadata retained
   - Ledger data: 7 years (regulatory requirement)
   - User data: Account lifetime + 7 years after closure
