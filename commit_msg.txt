feat(ussd-gateway): implement SMS fallback for completed transactions (Task 3.3.9 COMPLETE)

Features:
- ISmsService interface for SMS notification abstraction
- SmsService implementation with simulated SMS delivery (95% success rate)
- Multilingual SMS templates for payment receipts (5 languages: en, zu, xh, st, af)
- Fire-and-forget SMS sending after successful contribution payment
- Configurable SMS fallback via Sms:Enabled setting in appsettings.json
- SMS receipts include: group name, payment amount, and receipt number

Integration Points:
- Injected ISmsService into UssdMenuService for SMS sending capability
- SMS triggered after successful ProcessContributionAsync API call
- SMS sent asynchronously (non-blocking) using Task.Run to avoid delaying USSD response
- Utilizes session context (selected_group_name) and language preference for personalized messages

User Experience:
- User completes contribution via USSD
- USSD session ends with success message and receipt number
- SMS receipt arrives within seconds as backup confirmation
- SMS failure does not affect payment success or USSD flow
- User can view receipt in mobile app even if SMS fails

Limitations:
- Simulated SMS provider for MVP (integrates with NotificationService in production)
- SMS templates are hardcoded (production will use template service)
- No retry logic for failed SMS delivery (fire-and-forget pattern)
- SMS rate limiting not implemented (deferred to Phase 4.3.3)
